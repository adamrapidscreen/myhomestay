# MyHomestay Project Agent Context

This file is the repo-local doctrine for MyHomestay.

## Current Product Truth

MyHomestay is being treated as a greenfield project with the product spine, product brief, PRD-Lite, brand/UX system, architecture blueprint, and MVP build spec drafted.

The previous quick Vibe-session website was intentionally deleted from the working tree. Use Git history only if recovery is needed.

## Product Thesis

MyHomestay is an owner-first, Malaysia-first homestay platform for local operators who need better discovery, trust, and simple owner tools.

Travellers should discover trustworthy, locally contextual, Muslim-friendly homestay listings, then continue directly to the owner or admin on WhatsApp for booking and payment in the free MVP.

## Target Users

Primary first user:

- Homestay owners and small local operators in Malaysia.

Secondary users:

- Travellers looking for trusted local stays.
- Owner admins or family members who help manage inquiries.

Future expansion:

- Nearby Southeast Asian countries after Malaysia-first patterns are proven.

## Brand Direction

- Minimal
- Artisan
- Kampung modern
- Trustworthy
- Smooth and easy for owners and customers
- Not an Airbnb clone

## Commercial Direction

Free at first to build supply, trust, and usage.

Potential paid subscription later:

- Host premium features
- Booking management
- Verified online owner profile card
- Featured listings
- Monthly income tracker
- Analytics and multi-property tools

## Technical Direction

Target product shape:

- Responsive web app with hybrid PWA path
- Owner onboarding first
- Public listing pages
- WhatsApp booking/payment handoff for MVP
- Subscription architecture planned but not necessarily active in MVP
