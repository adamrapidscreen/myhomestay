# Beta 0.16 - Chapter 5 Launch Readiness

Date: 2026-06-01
Version: `0.16.0`

## Summary

Beta 0.16 closes Build Chapter 5 for a controlled-pilot baseline. The app now
has the minimum operator controls, durable public engagement metrics,
SEO/share metadata, and a file-backed pilot checklist.

This is still not a production-readiness claim. Payments, subscriptions,
verification policy, image moderation, and deployment gates remain outside this
release.

## Added

- Admin restore-live action for paused or needs-review listings.
- Migration `0011_chapter5_admin_restore_live.sql` for restore-live moderation
  RPC support and audit action constraint coverage.
- Durable metrics API at `/api/listing-metrics`.
- Listing view and WhatsApp click metric beacons.
- Directory and listing canonical/Open Graph/Twitter metadata.
- Dynamic listing OG image route at `/listings/[slug]/opengraph-image`.
- Chapter 5 pilot checklist at `docs/product/chapter-5-pilot-checklist.md`.
- Launch-bloom draft spec at `docs/product/launch-bloom-spec.md`.

## Changed

- `/admin` copy now reflects pause, restore, flag, and clear actions.
- Admin action parser and moderation data layer now include the `publish`
  restore-live action.
- Project version marker moved to Beta 0.16 / `0.16.0`.

## Known Residual Risks

- Public metrics can be inflated by repeat requests. Acceptable for MVP owner
  signals; not acceptable for billing, ranking, or paid placement.
- First-admin bootstrap currently requires a one-time SQL trigger
  disable/promote/re-enable sequence.
- Image moderation and owner/property verification policy are not implemented.
- WhatsApp booking and payment remain outside the platform.
- `npm audit` still reports known moderate Next/PostCSS transitive advisories
  where the available forced fix is breaking; no high/critical audit issues
  are present.
