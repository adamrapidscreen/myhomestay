# MyHomestay Beta 0.10 — Foundation and Owner Workflow

Date: 2026-05-28
Version: `0.10.0`
First release after the planning pack and KD Plan & Build pilot.

## What This Release Is

This is the first deployable release of the new MyHomestay greenfield product. The previous Vibe-session site has been intentionally deleted from the working tree.

It ships:

- A clean Next.js 15 App Router foundation with Tailwind v4 brand tokens.
- A real public surface: home, public directory with filters, public listing detail.
- A real owner workflow: dashboard, onboarding, listing builder, draft/publish/pause status transitions.
- A KD Plan & Build flow tracker with explicit Build Chapters, Work Cards, and verification receipts.

This release is **mock-data-only**. Supabase auth, real database, and real photo storage land in Build Chapter 4. Do not treat this release as production-ready.

## Highlights

### Foundation (Build Chapter 1)

- Next.js 15.5 + React 19 + Tailwind v4 + TypeScript 5.7.
- Kampung Quiet Ledger Hybrid brand tokens: paper, rice, stone, ink, leaf, river, clay, sun, danger.
- Fraunces (display) + Inter (body) via `next/font`.
- Strict CSS-first tokens via Tailwind v4 `@theme`.
- Typed listing, owner, photo, and Malaysian location contracts.
- Helpers: WhatsApp URL builder, Malaysian location formatter, listing completeness evaluator.

### Public Surface (Build Chapter 2)

- `/` owner-first home with sample listing teaser and 3-step explainer.
- `/listings` public directory with state, capacity, price, Muslim-friendly, family-friendly, and free-text filters via URL search params.
- `/listings/[slug]` public listing detail with gallery, trust strip, owner card, and WhatsApp CTA.
- Sticky mobile WhatsApp CTA on listing detail.
- Generates static pages for published listings.
- Non-published listings 404 via `notFound()`.

### Owner Workflow (Build Chapter 3)

- `/dashboard` owner workspace with status badges, completeness summary, photos / capacity / views / WhatsApp metrics, and decided next-action button per listing.
- `/dashboard/onboarding` server-action form: display name, full name, WhatsApp E.164 validation, and explicit consent that booking and payment continue on WhatsApp.
- `/dashboard/listings/new` and `/dashboard/listings/[id]/edit` listing builder with sections for basics, location, capacity and price, photos (placeholder categories), amenities, trust, and house rules.
- Server actions enforce: free tier limit (3 listings), owner scope (cannot edit another owner's listing), publish completeness gate, and status transition matrix.
- 4 listing statuses: `draft`, `published`, `paused`, `needs_review`.
- Mock data session caveat surfaced in the dashboard footer and onboarding hint.

### Toolchain

- Migrated from `next lint` to ESLint CLI flat config (`eslint .`). `next lint` is removed in Next 16.
- Vitest 4.1 with `jsdom` and `@/*` alias.
- 36 unit tests across `applyListingFilters`, `parseListingFilters`, `buildListingWhatsappUrl`, `evaluateListingCompleteness`, and related helpers.
- Reusable Atelier-47 visual review script at `.tmp/atelier/review.mjs` (gitignored). Runs Playwright Chromium across the public and owner surface at 320 / 375 / 768 / 1280.

## Known Limits

- Mock-data-only. Owner edits reset on `next start` restart.
- Photos are placeholder category divs. Real upload waits on Supabase Storage in Build Chapter 4.
- WhatsApp click tracking is `console.debug` in dev only. Durable metrics arrive in Chapter 5.
- `needs_review` status is admin-only and not exposed in this owner UI.
- `next lint` deprecation warning is gone after the ESLint CLI migration.
- 2 moderate-severity npm audit advisories remain. They are transitive and accepted for now. Review under the Chapter 4 security gate.

## Boundaries Held In This Release

- No "verified" language anywhere in the public surface.
- Booking and payment boundary copy visible on home, directory, listing detail, and onboarding.
- No payment, subscription, or checkout code.
- No real auth.
- No external dependencies beyond standard Next.js, React, Tailwind, ESLint, Vitest, and Playwright (dev only).
- No secrets, API keys, or environment-specific values committed to the repo.
