# Beta 0.15 — Chapter 4: Supabase Auth, Data, Storage

Date: 2026-05-29
Version: `0.15.0` (previous: `0.10.0`)

## Summary

MyHomestay moves from mock-data-only to a real Supabase backend: email OTP
auth, RLS-enforced data ownership, and a storage policy for listing photos.
The whole chapter was driven by a front-loaded Sentinel-47 Security Gate, then
reviewed by Sentinel-47 + Agent 47 Review Gate before this release.

## What landed

### Security Gate (front-loaded)
- `_planning/security-gate-chapter-4.md`: threat model (T1–T12, OWASP-mapped),
  schema, RLS intent, DB-layer business rules, storage policy, env strategy.

### Database (8 migrations)
- Schema: 5 RLS-enabled tables (`profiles`, `listings`, `listing_photos`,
  `listing_metrics`, `admin_review`) + enums + indexes.
- RLS: owner-scoped CRUD, admin gating via `is_admin()`.
- DB-enforced business rules (not just UI): 3-listing free-tier trigger,
  publish-completeness gate, status-transition guard, role-escalation guard.
- Storage: private `listing-photos` bucket (mime allowlist, 5MB cap),
  path-scoped object RLS.
- Auth bootstrap: auto-create profile on new user.
- Column-scoped public reads via `SECURITY DEFINER` RPCs (no postcode/PII leak).

### App
- Email OTP login (`/login`), session-refresh middleware, `requireOwner`
  route protection, sign-out.
- RLS-bound data layer (`src/server/listings-data.ts`) replacing mock stores.
- Storage helper with mime/size/path validation (upload UI deferred).
- Public surface reads via Supabase; listing detail is now dynamic.
- Owner components narrowed to a safe `PublicListingOwner` shape.

## Security review outcome

Sentinel-47 + Agent 47 Review Gate, evidence-based, against the live worktree:

- 0 critical, 0 high.
- M1 (postcode public leak) — fixed: app nulls postcode on public reads +
  migration `0008` removes anon table policies and routes public reads through
  column-scoped RPCs.
- M2 (publish gate violable post-publish) — fixed in migration `0007`.
- I1 (2 triggers missing locked search_path) — fixed in `0007`.
- L1 (admin column-scope) — deferred to Chapter 5; gate doc corrected.
- L2/I2/I3/I4 — accepted/documented residuals.

## Verified live (real Supabase)

- OTP request → emailed code → verify → session → dashboard.
- Profile save (RLS `profiles` UPDATE).
- Listing create as draft (RLS `listings` INSERT, owner-scoped).
- Publish gate passes a complete listing; draft → published transition.
- Public directory + home featured read through the `0008` column-scoped RPC.

4 runtime bugs found and fixed during live testing: duplicate hidden email
field, 6-vs-8 digit OTP length, copy string, `EMPTY_STATE` non-async export.

## Known limits / not done

- Cross-owner RLS denial not yet runtime-tested (needs 2-account browser test).
- M1 raw-API fix verified in code + app; raw-PostgREST confirmation pending.
- Admin moderation surface (pause / needs_review) is Chapter 5.
- 2 moderate transitive npm advisories (postcss under Next) accepted.
